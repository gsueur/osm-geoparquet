1 · Setup

Install extensions once per session. The url() macro keeps the snippets below readable.

INSTALL httpfs; LOAD httpfs;
INSTALL spatial; LOAD spatial;

CREATE OR REPLACE MACRO url(state, theme) AS
  'https://parquetry.geomermaids.com/latest/country=' ||
  split_part(state, '-', 1) || '/state=' || state || '/' || theme || '.parquet';

2 · Count features in a bounding box

Downtown Manhattan, roughly Times Square to Central Park — no full file download, just the bbox-pruned row groups:

SELECT COUNT(*) AS buildings
FROM read_parquet(url('US-NY', 'buildings'))
WHERE ST_Intersects(geometry, ST_MakeEnvelope(-74.00, 40.74, -73.96, 40.77));

3 · Filter + projection pushdown

Tall residential towers in Midtown, pulling only four columns:

SELECT name, building, levels, height
FROM read_parquet(url('US-NY', 'buildings'))
WHERE building = 'residential'
  AND levels > 30
  AND ST_Intersects(geometry, ST_MakeEnvelope(-73.99, 40.76, -73.95, 40.79))
ORDER BY levels DESC
LIMIT 10;

4 · Cross-theme spatial join with ST_Contains

Match each restaurant to the building it sits inside (Massachusetts):

SELECT b.name AS building, p.name AS restaurant, p.cuisine
FROM read_parquet(url('US-MA', 'buildings')) b
JOIN read_parquet(url('US-MA', 'pois')) p
  ON ST_Contains(b.geometry, p.geometry)
WHERE p.amenity = 'restaurant'
  AND b.name IS NOT NULL
LIMIT 10;

5 · Nearest features with ST_DWithin

Amenities within ~500m of a point, sorted by distance. Because rows are Hilbert-ordered, the row-group bbox filter trims almost everything before distance math runs:

WITH origin AS (SELECT ST_Point(-73.9857, 40.7484) AS pt)  -- Times Square
SELECT p.amenity, p.name, ST_Distance(p.geometry, o.pt) AS dist
FROM read_parquet(url('US-NY', 'pois')) p, origin o
WHERE ST_DWithin(p.geometry, o.pt, 0.005)  -- ~500m in degrees
ORDER BY dist
LIMIT 15;

6 · Aggregate across multiple states

Count POIs for a set of states in one query by passing a list of URLs:

SELECT state_iso, COUNT(*) AS pois
FROM read_parquet([
  url('US-NY', 'pois'),
  url('US-MA', 'pois'),
  url('US-CT', 'pois'),
  url('US-RI', 'pois')
])
GROUP BY state_iso
ORDER BY pois DESC;

7 · Python / GeoPandas

Same data, same URL, idiomatic from Python:

import geopandas as gpd

gdf = gpd.read_parquet(
    "https://parquetry.geomermaids.com/latest/country=US/state=US-RI/buildings.parquet"
)
print(f"{len(gdf):,} Rhode Island buildings")
print(gdf[["building", "name", "height"]].head())

What's in each file

Every parquet has this base schema plus theme-specific typed columns promoted from OSM tags:

osm_id      BIGINT
osm_type    VARCHAR    -- 'node' | 'way' | 'relation'
country     VARCHAR    -- 'US', 'CA', 'MX'
state       VARCHAR    -- e.g. 'New York', 'Ontario'
state_iso   VARCHAR    -- e.g. 'US-NY', 'CA-ON'
<typed columns>        -- per theme; see below
tags        MAP<VARCHAR, VARCHAR>
geometry    GEOMETRY   -- native Parquet logical type (GeoParquet 2.0)

Technical specs

  • GeoParquet 2.0 with the native Parquet GEOMETRY logical type (not WKB + sidecar metadata).
  • Hilbert-ordered rows for spatial clustering; row groups skip efficiently on bbox predicates.
  • ZSTD compression, 50,000 rows per row group — tuned for HTTP range-request reads.
  • Typed columns promoted from the OSM tags map; full map still present for any unpromoted keys.
  • Hive partitioning by country= and state= so engines can prune entire regions from a query.
  • Daily snapshots, immutable once published; latest/ alias points at the most recent.
  • Open-source pipeline (MIT).
